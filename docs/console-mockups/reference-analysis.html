Reference Product Analysis

What to steal, what to skip, and what WhatSoup needs that none of them have.

1. Wati — Multi-Number WhatsApp Management

What they do well
  • Number picker sidebar — dropdown to switch between connected WhatsApp numbers, each tagged (Support, Sales, Marketing)
  • Unified contact list — contacts from all numbers in one list, tagged with source number
  • Per-number analytics — filter inbox analytics by specific number
  • Shared automations — rules/chatbots run across all connected numbers
What WhatSoup needs differently
  • Wati treats all numbers identically. WhatSoup has three different runtimes per line — can't use a uniform inbox
  • No concept of groups — flat number list. WhatSoup needs group hierarchy
  • No connection health monitoring — assumes cloud-hosted reliability
  • No agent/SDK concept at all
Steal: Number picker/filter pattern. Tag lines visibly. Per-line analytics filtering.
Skip: Flat structure. Uniform inbox assumption.

2. Respond.io — Unified Messaging Inbox

What they do well
  • Three-panel layout — sidebar (inboxes/teams/filters) | conversation list | chat view
  • Quick Actions on hover — manage conversations without opening them
  • Team inbox concept — conversations grouped by team, with counts
  • Conversation cards — show channel icon, last message, timestamp, assigned agent, labels
  • Real-time dashboard widgets — open, unattended, unassigned counts
What WhatSoup needs differently
  • Respond.io is conversation-centric. WhatSoup is line-centric — the line is the primary object, not the conversation
  • No concept of different processing pipelines per channel
  • Agent means "human agent" — WhatSoup's agent means "Claude SDK runtime"
  • Missing: connection management, auth lifecycle, runtime state
Steal: Three-panel inbox layout for the Inbox screen. Conversation cards with context. Quick actions on hover. Real-time count widgets.
Skip: Conversation-first navigation. Human agent assignment model.

3. Chatwoot — Open-Source Multi-Channel

What they do well
  • Live dashboard — real-time KPIs: Open, Unattended, Unassigned with agent avatars
  • Heatmap — busiest times of day over last 7 days
  • Agent workload view — conversations per agent in real-time
  • Mine / Unassigned / All tabs — simple but effective filtering
  • Labels and conversation tagging
What WhatSoup needs differently
  • Chatwoot is support-team focused — agent assignment, SLAs, CSAT. WhatSoup is single-operator fleet management
  • No connection/runtime monitoring
  • No concept of different processing modes per channel
Steal: Live KPI cards. Heatmap for activity patterns. Simple tab filtering.
Skip: Agent assignment model. Support-team workflows.

4. Uptime Kuma — Self-Hosted Monitoring

What they do well
  • Service list with status bars — left sidebar shows each monitor with a heartbeat strip (green/red blocks)
  • Detail panel — click a service → see response time, uptime %, cert expiry, response chart
  • Dark theme, ops-first — designed for monitoring, not marketing
  • Grouping — monitors can be organized into collapsible groups
  • Status badges — 100%, 0%, with color coding
What WhatSoup needs differently
  • Uptime Kuma monitors external services. WhatSoup monitors its own managed connections + runtimes
  • Need profile-aware detail panels, not just response time
  • Need inbox/messaging integration, not just uptime
Steal: Heartbeat status strips per line. Ops-first dark aesthetic. Collapsible groups. Detail panel on click. KPI cards with sparklines (Grafana style).
Skip: External monitoring model. Static status pages.

What None of Them Have (WhatSoup's Gap)

  • Profile-aware pipelines — no product treats different lines as fundamentally different runtimes
  • Agent SDK runtime monitoring — active sessions, turn counts, tool usage, budget tracking
  • Hybrid AI escalation — rule engine → API → SDK is a unique flow
  • Connection lifecycle management — QR auth, reconnection, multi-device state for self-hosted WhatsApp
  • Single-operator fleet model — not a team tool, not a SaaS platform, but a personal ops console

Synthesis: The Soup Kitchen Should Be...

From Uptime Kuma

Heartbeat strips per line. Dark ops aesthetic. Group → service hierarchy. Click-to-detail pattern.

From Grafana

KPI stat cards with sparklines at the top. Color-coded thresholds. Time-range selector for metrics.

From Respond.io

Three-panel inbox layout. Conversation cards. Quick actions on hover. Real-time count badges.

From Wati

Number/line picker with tags. Per-line analytics filtering. Unified contact view across lines.
